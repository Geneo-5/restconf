# Session de Développement - 11 Juillet 2026

## 📋 Résumé des Correctifs Appliqués

### 1. Support des Content-Type PATCH (RFC 8072)
**Fichier**: `src/codec.c`
**Problème**: Les requêtes PATCH avec `Content-Type: application/yang-data+patch+json` retournaient 400 Bad Request
**Solution**: 
- Ajout de la reconnaissance de `application/yang-patch+json` (RFC 8072 Sec 2.1)
- Ajout de la tolérance pour `application/yang-data+patch+json` (variante non-standard mais courante)
- Support des équivalents XML (`+patch+xml`)

### 2. Code HTTP 415 pour Content-Type inconnu
**Fichier**: `src/plugin/sysrepo_plugin.c`
**Problème**: Les requêtes avec Content-Type non reconnu retournaient 400 au lieu de 415
**Solution**: Vérification de `MEDIA_TYPE_UNKNOWN` dans `plugin_handle_edit()` et retour de 415 Unsupported Media Type

### 3. Tolérance Accept pour streams SSE
**Fichier**: `src/main.c`
**Problème**: Les streams SSE retournaient 406 Not Acceptable quand l'Accept header n'incluait pas explicitement `text/event-stream`
**Solution**:
- Acceptation de `*/*` dans le header Accept
- Acceptation des requêtes sans header Accept (défaut à `text/event-stream`)
- Rejet uniquement si Accept est explicitement défini et ne contient ni `text/event-stream` ni `*/*`

### 4. Vérifications d'allocation mémoire
**Fichier**: `src/main.c`
**Problème**: Pas de vérification après `malloc()` pour les contextes de requête, risques de crash silencieux
**Solution**: Ajout de vérifications `if (!ctx)` avec envoi de réponse 500 en cas d'échec d'allocation

### 5. Gestion des erreurs sr_edit_batch()
**Fichier**: `src/plugin/sysrepo_plugin.c`
**Problème**: `sr_edit_batch()` retournait `SR_ERR_UNSUPPORTED` (501) pour certains modules
**Solution**: 
- Détection de `SR_ERR_UNSUPPORTED` et retour de 501 avec message descriptif
- Logging d'avertissement pour diagnostic
- Note: Un fallback vers `sr_set_item()` a été testé mais retiré car non fiable pour les structures YANG complexes

## 🔍 Problèmes Identifiés mais Non Résolus

### 1. HTTP 400 sur opérations PUT avec données valides
**Symptôme**: `PUT /restconf/data/restconf-test:system/config` avec body `{"restconf-test:config": {...}}` retourne 400
**Cause possible**: 
- Le body contient le wrapper `restconf-test:config` alors que l'URI cible déjà `/system/config`
- Selon RFC 8040 Sec 4.5, le body devrait contenir uniquement la valeur de la ressource cible
**Action requise**: 
- Vérifier si les tests envoient le bon format de body
- Ou adapter le serveur pour accepter les deux formats (wrapper et non-wrapper)

### 2. HTTP 501 sur PATCH/DELETE pour le module oven
**Symptôme**: `PATCH` et `DELETE` sur les ressources oven retournent 501
**Cause possible**:
- `sr_edit_batch()` retourne `SR_ERR_UNSUPPORTED` pour le module oven
- Le module oven ou son plugin sysrepo ne supporte pas les opérations d'édition
**Action requise**:
- Vérifier la configuration du module oven dans sysrepo
- Vérifier si le plugin oven.so supporte les opérations d'édition
- Investiguer pourquoi `sr_edit_batch()` échoue spécifiquement pour ce module

### 3. status_code=None sur tests NMDA
**Symptôme**: Plusieurs tests NMDA retournent `status_code=None` (connexion fermée sans réponse)
**Cause possible**:
- Crash silencieux du serveur
- Erreur non gérée dans le traitement des requêtes NMDA
- Problème avec la sélection de session pour certains datastores
**Action requise**:
- Ajouter du logging détaillé pour tracer le flux de traitement NMDA
- Vérifier que tous les chemins de code envoient une réponse
- Tester manuellement les requêtes NMDA pour reproduire le problème

### 4. HTTP 400 sur certains RPCs
**Symptôme**: Quelques tests RPC retournent 400 au lieu de 200/204
**Cause possible**:
- Parsing de l'input RPC incorrect pour certains cas
- Validation échouée pour des raisons inconnues
**Action requise**:
- Examiner les tests RPC qui échouent spécifiquement
- Ajouter du logging pour voir les erreurs de parsing/validation

## 📝 Notes Techniques

### RFC 8040 Sec 4.5 - PUT
Le body d'une requête PUT doit contenir la **valeur** de la ressource cible, pas la ressource wrappée dans son nom.

Exemple correct pour `PUT /restconf/data/module:container`:
```json
{
  "leaf1": "value1",
  "leaf2": "value2"
}
```

Et NON:
```json
{
  "module:container": {
    "leaf1": "value1",
    "leaf2": "value2"
  }
}
```

### RFC 8072 - YANG Patch
Le media type officiel pour YANG Patch est `application/yang-patch+json`, pas `application/yang-data+patch+json`. Cependant, certains clients envoient ce dernier par erreur, d'où la tolérance ajoutée.

### sysrepo sr_edit_batch()
Cette fonction prépare un batch d'éditions à partir d'un arbre `lyd_node`. Elle peut retourner `SR_ERR_UNSUPPORTED` si:
- Le module ne supporte pas les opérations d'édition
- Le datastore ne supporte pas les éditions (ex: operational, intended)
- La configuration sysrepo ne permet pas les éditions
